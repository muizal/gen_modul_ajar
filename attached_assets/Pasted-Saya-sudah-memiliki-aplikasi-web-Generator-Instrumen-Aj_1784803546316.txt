TUJUAN TAHAP INI:
Menjadikan data Capaian Pembelajaran (CP) sebagai sumber utama seluruh dokumen dan mencegah aplikasi membuat CP/materi palsu ketika data tidak tersedia.

1. HAPUS SISTEM FALLBACK DATA GENERIK

Cari fungsi:

getSafeData(mapel, kelas)

Saat ini jika data tidak tersedia, fungsi membuat data seperti:

* Konsep Dasar
* Analisis Lanjutan
* Praktik Terapan
* Evaluasi dan Refleksi

Hapus perilaku tersebut.

Jika data mapel/fase tidak tersedia, return array kosong:

return [];

Aplikasi DILARANG membuat materi, TP, CP, atau kompetensi generik sebagai pengganti data yang tidak tersedia.

2. TAMBAHKAN VALIDASI DATA

Jika pengguna memilih mapel/kelas yang CP-nya belum tersedia, tampilkan pesan:

"Data Capaian Pembelajaran untuk mata pelajaran dan fase ini belum tersedia. Silakan hubungi administrator."

Jangan menghasilkan dokumen.

3. PISAHKAN CP DARI MATERI

Struktur data jangan lagi hanya:

topik
jp
tp

Persiapkan struktur agar nantinya dapat menggunakan:

mapel
fase
kelas
elemen
cp
tp
materi
jp
pertemuan
sumber
nomor_regulasi
tahun_regulasi

Tidak perlu mengisi semua CP sekarang.

Gunakan data yang sudah ada sementara, tetapi struktur kode harus siap menerima format tersebut.

4. BUAT FILE DATA TERPISAH

Pindahkan konstanta:

const cpData = {...}

dari index.html ke:

/data/cp-data.js

Kemudian load melalui:

<script src="/data/cp-data.js"></script>

sebelum script utama aplikasi.

Pastikan aplikasi tetap berjalan setelah pemindahan.

Jangan membuat database/server pada tahap ini.

5. TAMBAHKAN METADATA SUMBER CP

Setiap dataset CP nantinya harus dapat memiliki:

source: {
regulation: "",
number: "",
year: "",
status: "active"
}

Jangan mengarang nomor regulasi.

Jika belum tersedia gunakan string kosong.

6. BEDAKAN FASE DAN KELAS

Gunakan:

Kelas X = Fase E
Kelas XI = Fase F
Kelas XII = Fase F

Tetapi jangan menganggap CP kelas XI dan XII berbeda jika regulasi hanya mendefinisikan CP berdasarkan Fase F.

CP disimpan berdasarkan fase.

Materi/ATP boleh memiliki pemetaan kelas XI/XII.

7. VALIDASI SEBELUM GENERATE

Sebelum fungsi generateModul() menghasilkan dokumen, lakukan pemeriksaan:

* jenis dokumen tersedia
* mapel tersedia
* kelas tersedia
* fase dapat ditentukan
* data CP tersedia

Jika CP tidak tersedia:
STOP proses generate.

Jangan menggunakan fallback.

8. TAMBAHKAN FUNGSI HELPER

Buat:

getPhaseFromClass(kelas)

Return:
X -> E
XI -> F
XII -> F

Buat:

getCPData(mapel, fase)

Fungsi ini hanya mengambil data yang benar-benar tersedia.

Tidak boleh membuat data baru.

Buat:

hasCPData(mapel, fase)

untuk mengecek apakah CP tersedia.
